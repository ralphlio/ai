GitHub 入门

普通人也能看懂的 GitHub：从保存代码到发布网页

GitHub 可以先理解成一个“带历史记录的项目云盘”。你可以用它保存代码、管理网页、展示作品，也可以把一个简单网站发布到互联网上。

先理解几个词

不需要一开始就懂全部命令。先把下面几个词和生活里的概念对应起来，就能读懂大多数 GitHub 页面。

仓库

一个项目文件夹。一个网站、一个工具、一份电子简历，都可以放在一个仓库里。

提交

保存一个版本。它会记录谁在什么时候改了什么，方便以后回看或恢复。

推送

把本地修改上传到 GitHub。只提交不推送，别人还看不到你的新版本。

分支

项目的一个副本。适合试验新想法，确认没问题后再合并回主版本。

Pull Request

请求把某些修改合并进项目。团队通常用它来讨论和检查代码。

GitHub Pages

GitHub 提供的网页发布功能。它能把仓库里的 HTML 页面变成公开网址。

Git 和 GitHub 有什么区别

两者经常一起出现，但不是同一个东西。

Git

本地版本管理工具。它像“修订记录”，帮你记录每次改动。

GitHub

网上的项目托管平台。它像“云端项目空间”，让项目可备份、可协作、可展示。

新手实操：用 GitHub 发布自己的网页

如果你只想拥有一个个人主页或作品展示页，可以从 GitHub Pages 开始。

1. 注册账号 进入 github.com，创建一个 GitHub 账号。
2. 创建仓库 新建仓库，比如命名为 my-homepage。
3. 上传网页 准备一个 index.html，并上传到仓库。
4. 开启 Pages 在 Settings 里找到 Pages，选择 main 分支。
5. 访问网址 等待片刻，打开 GitHub 给出的网页链接。

一个最简单的网页示例

网页项目最少可以只有一个文件：index.html。

<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <title>我的个人主页</title>
  </head>
  <body>
    <h1>你好，我是 Ralph</h1>
    <p>这里展示我的作品和文章。</p>
  </body>
</html>

这里展示我的作品和文章。

日常更新网页的流程

发布之后，后续维护通常就是重复下面这套动作。

修改本地文件

比如更新文案、替换图片、调整颜色，先在电脑上的项目文件夹里完成。

提交修改

提交相当于保存一个新版本。提交信息要简短说明这次改了什么。

推送到 GitHub

推送后，远程仓库才会收到新代码。GitHub Pages 会根据新代码重新发布网页。

打开网页检查

Pages 有时会延迟几分钟。若网页没立刻变化，先刷新或稍后再看。

安全提醒

GitHub 很适合公开展示，但不是所有文件都应该放上去。

不要上传这些内容

密码、GitHub token、API key、私钥文件、个人身份证件、包含隐私的数据文件。

建议上传这些内容

网页代码、公开图片、项目说明、作品案例、不会泄露隐私的文档和配置示例。

常见问题

刚开始使用 GitHub 时，最容易卡在这些地方。

为什么我看到的是代码，不是网页？

你可能打开了 GitHub 仓库文件页面。网页要从 GitHub Pages 提供的网址访问。

为什么网页没有马上更新？

GitHub Pages 发布和浏览器缓存都可能有延迟。通常等一会儿再刷新即可。

公开仓库是不是所有人都能看？

是的。公开仓库适合作品展示，不适合存放密钥、密码或私人资料。

我一定要会命令行吗？

不一定。新手可以先用网页上传文件，之后再学习 GitHub Desktop、Codex 或命令行。

推荐学习路径

不用一次学完。先完成一个小网页，你就已经掌握了最重要的部分。

第一步

理解仓库、提交、推送和 GitHub Pages。

第二步

学会修改网页并反复发布更新。

第三步

再学习分支、Pull Request、多人协作和自定义域名。

GitHub 可以先从一个小网页开始学。能保存、能更新、能发布，就已经很实用了。
